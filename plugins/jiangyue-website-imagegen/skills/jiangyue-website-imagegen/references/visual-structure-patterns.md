# Visual Structure Patterns

Use this library when the image is high-impact, abstract, multi-subject, physically metaphorical, or has already failed for weak design. The patterns are starting points, not fixed templates.

Every structure must be written as:

1. **Relationship model:** how foundation, driver, outcome, and flow or feedback relate.
2. **Variable parameters:** what can change to fit the asset and avoid generic output.
3. **Forbidden boundary:** what the structure must not become.

Do not lock the whole skill to fan, HVAC, pump, or any one application. Use application context only when the page, product, or user brief makes it relevant.

## How To Use The Library

1. Start from the intent lock and choose the relationship model.
2. Select a structure family that can make that relationship visible.
3. Change variable parameters until the structure has a specific image role. Use a strong image visual hammer only when the image owns first attention.
4. Check forbidden boundaries before writing the production brief.
5. If the structure is reliable but dull, change the crop, scale, silhouette, material, or structure family before changing color or glow.

The structure should create freedom through controlled variables. It should prevent impossible or generic visuals without forcing every image into the same formula.

## Structure Families

### 1. Layered Control Core

**Relationship model:** engineered foundation supports an intelligent control core; service appears as access, protection, continuity, or feedback around the core.

**Variable parameters:**

- foundation carrier: PCB, engineered plate, schematic plane, test fixture, documentation plane, or abstract control architecture
- driver carrier: chip, module, decision layer, control-signal layer, or adaptive feedback cue
- service carrier: complete transparent cover, access edge, support envelope, feedback path, service touchpoint, or page copy
- possible image role: main visual hammer, professional atmosphere, brand technology mood, or trust-supporting hero layer
- possible image visual hammer: chip on disciplined board, control module under transparent cover, cutaway stack, close-cropped engineered object
- realism level: conceptual electronics, clean 3D product fragment, deterministic SVG, or mixed image plus typeset overlay

**Forbidden boundary:**

- partial transparent cage, half shell, floating frame, arbitrary metal base, or cover with no thickness
- blue light treated as impossible electricity outside a device
- generic AI icon pasted onto hardware
- PCB traces, labels, or components implying unverified final product architecture

### 2. Controller-To-Application System

**Relationship model:** controller acts on an application; sensing or operating feedback returns to the controller.

**Variable parameters:**

- application carrier: OEM equipment, fan, HVAC unit, pump, ventilation scene, machine fragment, or abstract load only when relevant
- controller carrier: product-like control unit, PCB fragment, module, control layer, or page-copy mention
- flow carrier: control signal, data path, feedback path, airflow relationship, or sequence line
- crop: close system fragment, left-right relationship, foreground-background depth, or minimal schematic composition
- context depth: clean studio, application environment fragment, technical bench, or abstract system space

**Forbidden boundary:**

- unverified customer installation or final equipment integration
- decorative energy loop with no source, load, wiring, or safety boundary
- application object overpowering the controller when the page is about Jiangyue capability
- forced fan/HVAC/pump imagery when the current page is broader

### 3. Engineering Workflow

**Relationship model:** buyer requirement or technical input moves through engineering analysis, control logic, and support handoff.

**Variable parameters:**

- input carrier: specification sheet, wiring diagram, product photo fragment, abstract requirement blocks, or test data shape without readable claims
- driver carrier: engineering workspace, control algorithm layer, AI-assisted analysis cue, or structured decision path
- outcome carrier: refined controller concept, documented support route, selected parameter set, or inquiry-ready handoff
- possible image role: professional atmosphere, process credibility, inquiry trust support, or main visual hammer
- possible image visual hammer: precise desk/bench composition, layered documents and hardware, clean workflow axis, or annotated object without fake details

**Forbidden boundary:**

- handshake, call-center headset, smiling stock office, or generic service desk
- readable fake measurements, certificates, UI values, or compliance marks
- cluttered engineering scene that feels busy rather than disciplined

### 4. Support Envelope

**Relationship model:** service does not sit beside the technology; it protects, maintains, and keeps the technical core accessible over time.

**Variable parameters:**

- envelope carrier: complete transparent cover, service layer, access panel, maintenance path, documentation band, or response field
- core carrier: controller, PCB, module, control architecture, or application system
- continuity cue: feedback line, looped path, support boundary, repeated touchpoints, or page-copy support promise
- material: transparent polycarbonate-like cover, light technical glass, matte shell, or abstract boundary

**Forbidden boundary:**

- half frame, decorative halo, floating protective ring, or enclosure that could not be manufactured
- service represented only by a generic icon
- shell hiding the image's intended subject or making the support atmosphere visually muddy

### 5. Diagnostic Feedback Loop

**Relationship model:** sense, analyze, adjust, and return feedback. Use when AI, diagnostics, adaptive control, or long-term service must be shown as behavior.

**Variable parameters:**

- sensing carrier: abstract sensor point, signal trace, application feedback, or inspection cue
- analysis carrier: chip, control layer, decision node, or abstract AI field
- action carrier: controller output, parameter adjustment, refined path, or application response
- loop form: circular return, U-shaped path, layered cycle, or linear path with return channel

**Forbidden boundary:**

- calling the path energy flow unless a plausible power system is shown
- dashboard screens with fake values
- decorative circular arrows that do not connect to real carriers
- excessive glow or sci-fi motion that weakens industrial credibility

### 6. Abstract Control Architecture

**Relationship model:** capability relationship is shown through planes, axes, hierarchy, and flow rather than literal hardware.

**Variable parameters:**

- foundation: stable plane, grid discipline, engineering baseline, dense-lower/simple-upper hierarchy
- driver: active control layer, bright path, adaptive node, asymmetric movement, or central transformation
- outcome: resolved object, clear endpoint, protected zone, or typographic anchor
- medium: SVG/canvas, clean 3D abstract image, or hybrid with typeset text

**Forbidden boundary:**

- three equal cards, equal icons, default hexagons, generic network nodes, or stock AI poster
- visual that would still fit any software company if Jiangyue's motor-control context were removed
- structure that needs a paragraph of explanation to be understood

### 7. Product Or Technology Fragment

**Relationship model:** one credible detail represents the larger engineering capability; page copy carries the broader claim.

**Variable parameters:**

- fragment: heat-sink rhythm, connector edge, PCB corner, enclosure seam, fastener detail, control module, or thermal path
- crop: macro, edge cut, low angle, top-down disciplined geometry, or partial cutaway
- message carrier: material credibility, precision, scale, restrained signal cue, or HTML text
- background: clean industrial surface, light technical space, or application-adjacent context

**Forbidden boundary:**

- pretending the fragment is the final product if no verified product photo exists
- invented labels, ports, certifications, or specs
- fragment so abstract that the buyer cannot connect it to motor control or industrial equipment

## Optimization Levers

Use these levers to raise impact without breaking reliability:

- change relationship model: foundation-driver-outcome, controller-application-feedback, workflow, support envelope, diagnostic loop
- change crop: tighter, off-center, edge interaction, object larger than frame, or deliberate negative space
- change silhouette: one compact symbol, layered stack, protected core, directional system, or cutaway form
- change carrier: move a subject from icon to material, page copy, path behavior, or physical layer
- change medium: image generation for atmospheric realism, SVG/canvas for precise geometry and text, hybrid for hero polish
- change specificity: broad industrial control for general pages, application-specific cues only when relevant

Do not solve weak design by only changing palette, shadows, lens flare, glow, or background gradients.
