# Daily Curation Rules

Use daily curation when the user asks to organize today's work, summarize useful task outputs, or prepare a short list of items for confirmation.

## Scope

Prefer current project files and the current conversation. Do not assume access to all Codex threads unless the user provides links, files, or a thread-management tool is available and explicitly requested.

## Daily Output

Write daily summaries to:

```text
brand-system/91-daily-curation-log/
```

Daily summaries may mention inbox candidates, but they are not formal knowledge authority.

## Minimal Daily Summary

```text
# YYYY-MM-DD Daily Curation

## Confirmation Needed

## Added To Formal Knowledge

## Inbox Candidates

## Process-Only Archive

## Product Fact Risks

## Follow-Up Questions
```

Do not add unconfirmed materials to formal knowledge just because they appeared in a daily summary.
